# crypto-AI

Монорепозиторий проекта crypto-AI.

Проект объединяет:
- backend API
- web-приложение
- Telegram-бота
- общую логику анализа рынка
- документацию и командные процессы

## Назначение проекта

crypto-AI — система для анализа крипторынка, расчёта торговых сигналов, отображения данных в web-интерфейсе и интеграции с внешними поставщиками рыночных данных.

## Состав проекта

- `apps/api` — backend API, бизнес-логика, интеграции
- `apps/web` — frontend
- `packages` — общие пакеты и shared code
- `docs` — документация проекта
- `.github` — шаблоны, workflow, правила работы в GitHub

## Основные сценарии

- получение рыночных данных
- расчёт сигналов buy / sell / hold
- отображение аналитики в web dashboard
- работа Telegram-бота
- документирование требований, бизнес-правил и архитектуры

## Карта документации

- `CONTRIBUTING.md` — правила работы с репозиторием
- `docs/processes/github-workflow.md` — GitHub workflow команды
- `docs/architecture/README.md` — архитектура
- `docs/api/README.md` — API
- `docs/business-rules/README.md` — бизнес-правила
- Wiki — база знаний, onboarding, glossary, FAQ

## GitHub workflow

Основной поток:
Issue → Branch → Commit → Pull Request → Review → QA → Merge → Docs update

## Статусы задач

- `Backlog` — задача создана, но ещё не готова к работе.
- `Ready for Analysis` — нужен анализ, уточнение требований, AC, зависимости.
- `Ready for Dev` — задача проработана, можно брать в разработку.
- `In Progress` — задача в активной работе.
- `In Review` — открыт PR, идёт review.
- `Ready for QA` — изменения смержены, можно проверять.
- `In QA` — тестирование в процессе.
- `Done` — задача завершена и подтверждена.
- `Blocked` — есть блокер, причина обязана быть зафиксирована.

## Как начать работу

1. Склонировать репозиторий
2. Установить зависимости
3. Настроить `.env`
4. Запустить локальные сервисы

```bash
pnpm install
pnpm dev
